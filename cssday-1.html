In and Out Of Style Why standards matter

Jeremy Keith
Founder / Developer @ Clearleft

Jeremy heeft al eens eerder op het podium van een CSSday gestaan, dit keer trapt hij de editie van 2022 af. In zijn sessie neemt hij ons mee in een inspirerend verhaal over de geschiedenis van het World Wide Web. In zijn verhaal is de kern van de boodschap dat gebruikers resistent zijn tegen veranderingen. Toch is er in de loop der jaren al veel veranderd. Hierbij is padafhankelijkheid bij komen kijken, wat betekend dat bestaande methoden werden gebruikt om het begrip te behouden en dat er een gedeelde context werd gecreëerd. Vandaar dat herkenbaarheid belangrijk is, want mensen zijn gewoontedieren, ook al wordt deze langzaam aan veranderd.

Hoe styling evolueerde

Stapsgewijs neemt Jeremy mee aan de hand van een aantal slides hoe HTML en CSS is geevolueerd naar wat het nu is. Het begon ooit met een beperkte set aan HTML elementen als <font>, <big> en <center> en de declaratie van stijling wat meer leek op JavaScript syntax. Door de doorontwikkelingen en de kleine wijzigingen daarbij telkens is het de HTML en CSS van tegenwoordig wat veel elementen, eigenschappen en declaraties kent.

De toekomst

  • Met Cascade Layers @layer is een toevoeging aan de CSS Specificity om er meer invloed op te hebben.
    The CSS Cascade, A Deep Dive door Bramus van Damme
  • Met Scoping @scope richt zich op een substructuur of fragment van een webpagina dat met selectors gericht gebruikt kan worden voor de declaratie van de stijl.
    The Future of Organizing Your CSS door Tab Atkins-Bittner

Wat ontbreekt er nog?

Het stijlen van formulier bedieningselementen is en blijft lastig, met name bij datumkiezers, selecties en kleurkiezers. Hiervoor heeft Jeremy wel een tip, kijk eens naar het project open-ui.org. Het doel van dit project is om ingebouwde besturingselementen te stylen en uit te breiden. Webstandaarden in de browser vormen de basis om op voort te bouwen, het is uniform en daardoor herkenbaar.

Interop 2022 A commit to improve interoperability

Rachel Andrew
Technical Writer @ Google LLC / web.dev

Interop 2022 is een samenwerking tussen de grote browser ontwikkelaars Apple, Google, Microsoft en Mozilla met daarnaast twee softwareadviesbureaus Bocoup en Igalia. Het doel van deze samenwerking is om voor consistenter web te zorgen door te gezamenlijk focussen op dezelfde (door)ontwikkelingen van webtechnieken.

Interop 2022 / Compat 2021: 5 Focus Areas afgerond

  1. aspect-ratio: om de beeldverhouding van het element te bepalen, nooit meer padding hack!
  2. gap: voor Flexbox is geintroduceerd.
  3. self-start; en self-end; als properties voor rtl ltr uitlijning richting voor Flexbox en Grid.
  4. position: sticky; om een element te laten plakken, eindelijk geen JavaScripts meer nodig.
  5. transform: voor betere ondersteuning van rotate(), scale(), skew() en translate().

Interop 2022: 10 Focus Areas in (door)ontwikkeling

  1. Met Cascade Layers @layer is een toevoeging aan de CSS Specificity om er meer invloed op te hebben.
    The CSS Cascade, A Deep Dive door Bramus van Damme
  2. Color Spaces and Functions color-mix(); en color-contrast();.
  3. Containment contain: voor het isoleren van de styling tot een bepaald element op de pagina wat ten goede komt aan de prestatie van de pagina.
  4. <dialog> voor een native dialog/modal inclusief achtergrond styling en een JS API om het te openen of te sluiten.
  5. accent-color: voor formulier elementen om in een keer de accent kleur toe te passen.
  6. overscroll-behavior: en scroll-snap voor scrolling. Het einde is nabij voor sliders met (veel) JavaScript.
    Oh Snap! That Scroll Control door Adam Argyle
  7. subgrid; eindelijk de optie om de grid styling te overerven van parent naar child grid.
    Creative CSS Layout door Michelle Barker
  8. Viewport Units: dvh dynamic viewport height, lvh largest viewport height en svh smalled viewport height. Voor betere ondersteuning van de wisselende viewport op mobiele devices.

Voortgang van de Focus Areas

De voortgang van de Focus Areas is op het Interop 2022 Dashboard te zien, de scores per browser op het dashboard worden door Web Platform Tests berekend.

Bijdrage leveren aan het testen en Interop 2023

Er is er nog genoeg om te testen en verbeteren. Iedereen kan een bijdrage leveren aan het testen door bugs te melden in het Interop 2022 project op Github, zo heb je indirect invloed op verbeteringen. Of als je ideeën hebt waar Interop 2023 zich op moet focussen dan kan je dit aangeven.

The CSS Cascade A deep dive

Bramus van Damme
Chrome DevRel Engineer @ Google LLC

Met Bramus namen we een diepe duik in CSS Specificity en wat de toevoeging van het nieuwe Cascade Layers betekend en hoe het toegepast kan worden.

Wat is CSS Specicifity

De CSS Specificity is het algoritme dat door browsers wordt gebruikt om de CSS-declaratie te bepalen die het meest relevant is voor een element, wat op zijn beurt de eigenschapswaarde bepaalt die op het element moet worden toegepast. Het algoritme berekent het gewicht van een CSS-selector om te bepalen welke regel van concurrerende CSS-declaraties op een element wordt toegepast.

Introductie van Cascade layers

Door het gebruiken van specifieke pseudo-elements als :is(), :has(), :where() en :not() of het gebruiker van standaard stijling kan het een behoorlijke uitdaging zijn om stijling te overschrijven. Met Cascade Layers als toevoeging aan de CSS Specificity wordt het mogelijk om styling aan de hand van @layer te ordenen en een hogere of lagere prioriteit te geven. Dit maakt het makelijker om overschrijven van eerder vastgelegde stijling.

                    @layer reset, base, components, utilities;
                
                    
                        @layer reset {
  
                            ...

                        }



                        @layer base {

                            ...

                        }



                        @layer components {

                            ...

                        }



                        @layer utilities {

                            ...

                        }
